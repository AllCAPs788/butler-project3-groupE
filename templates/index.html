Recommender Systems

Recommender systems are everywhere on the web: suggesting new media content you might like such as videos or articles, products to buy on e-commerce sites, and even new people to meet on dating apps.

But how do they work?

Collaborative vs. Content-Based

Many recommender systems make use of collaborative filtering, content-based filtering, or a combination.

Collaborative filtering needs data about the users to work, it recommends items based on similarities between users. An example is if person A and person B have both watched the same movie, a different movie that B watched could be recommended to A. The system doesnâ€™t need to know anything about the movies, only who watches them. An algorithm that could be used for collaborative filtering is k-nearest neighbor.

Content-based systems use information about the items, not the users. This is useful when user data is not stored by the site or does not require a login. An example is if a user listens to a song, the system will compare the attributes of that song to others and recommend similar songs.

Our Model

Our model is a content-based system which uses attributes of a book to give recommendations to the user for similar books.

The attributes used are: title, author, publisher, and category. The TF-IDF algorithm is used to analyze the text of these book attributes and creates a matrix. The user chooses one book, and the system will use the matrix to find the 10 most similar books to the one they chose. Go to "Our Model" to try it out yourself!
